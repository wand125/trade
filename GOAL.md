# XAUUSD Short-Term Trade Prediction Research Goal

## 目的

XAUUSD の短期チャートデータを使い、任意の 1 か月間における利益最大化を目指すトレード予測プログラムを研究・開発する。

本プロジェクトは、実運用を直ちに目的にするものではなく、再現可能な研究基盤、検証可能なバックテスト、過学習に強いモデル評価、継続的な実験記録を整えることを第一目標とする。

## 取引ルール

- 対象銘柄は XAUUSD。
- 1 回の取引サイズは固定で 1 オンス。
- 取引方向はロングまたはショート。
- 同時に保持できる玉は 1 つまで。
- ポジションを決済するまで、次の新規エントリーはできない。
- エントリーと決済は、許される任意のタイミングで何度行ってもよい。
- ポジションは最大 24 時間まで保持できる。
- 24 時間以内に決済されなかったポジションは強制決済する。
- 現行の研究標準では、利益は 1.0 倍、損失は 1.20 倍として評価する。
- 初期仕様の 0.9 / 1.3 は過去比較用として残し、使う場合は明示する。

暫定的な損益定義:

```text
raw_pnl = 方向付き価格差 * 1オンス
adjusted_pnl = raw_pnl * 1.0  if raw_pnl > 0
adjusted_pnl = raw_pnl * 1.2  if raw_pnl < 0
adjusted_pnl = 0              if raw_pnl = 0
```

この損益定義を現行の標準評価条件とし、Bid/Ask、スプレッド、スリッページ、手数料を導入する段階で再確認する。

## 最適化対象

主目的は、任意の 1 か月間における `adjusted_pnl` の最大化。

ただし単月の最大利益だけを追うと過学習しやすいため、以下も同時に評価する。

- 複数月での平均利益
- 月次利益の安定性
- 最大ドローダウン
- 取引回数
- 勝率
- Profit Factor
- 1 取引あたり期待値
- 保有時間分布
- ロング/ショートの偏り
- データ期間を変えたときのスコア劣化

## 研究要件

- 差分、リターン、微分、FFT、周波数領域特徴、ボラティリティ、時間帯特徴などの前処理を検証する。
- 深層学習によって、エントリータイミング、方向、決済タイミングを推定する。
- レイヤー数、ニューロン数、系列長、損失関数、最適化手法、正則化を体系的に調整する。
- 学習途中でも、ベンチマークスコア、検証スコア、モデルチェックポイント、設定、ログを保存する。
- 過学習を避けるため、時系列分割、walk-forward 検証、期間外テスト、リーク防止を必須にする。
- データ欠損、週末/祝日ギャップ、ノイズ、異常スプレッドに対応し、ロバスト性を高める。
- 研究の途中から再開しても、何をすべきか分かる状態を保つ。
- プログラム、設定、実験結果、判断理由を残す。
- 必要なライブラリ、外部データ、既存手法は適宜調査し、採用理由と検証結果を記録する。

## ドキュメント運用

研究は次の 2 系統で記録する。

- 時系列レポート: 日々の作業、実験、気づき、失敗、次の行動を `docs/research_log.md` に追記する。
- 体系的まとめ: 仕様、評価方法、データ方針、モデル方針、アイデア、意思決定を `docs/` 配下に整理する。

常に以下を満たす状態を目指す。

- 最新の研究状態は `docs/status.md` を読めば分かる。
- 次にやる作業は `docs/status.md` と `docs/research_plan.md` に書かれている。
- 実験の設定と結果は `docs/reports/` に残る。
- 仕様変更や重要判断は `docs/decisions/` に残る。
- 未検証アイデアは `docs/ideas.md` に集約する。
