## Summary

- Experiment ID: `holding_max_grid_2025_01_08`
- Status: `250..260m` band promoted for the next fixed check; not yet standard policy
- Main result: broad 2025-01..08 verification changed the conclusion from a single `240m` candidate to a more stable `250..260m` band.
- Current robust candidate: `max_predicted_hold_minutes=260`.

## Setup

- Baseline policy source: existing `stateful_p5` timed-EV config.
- Prediction input: stitched validation OOF predictions for 2025-01..05 plus fixed apply predictions for 2025-06..08.
- Evaluation months: 2025-01..2025-08.
- PnL adjustment: profit multiplier `1.0`, loss multiplier `1.20`.
- Cost cases:
  - `nocost`: spread `0.0`, slippage `0.0`, execution delay `0`
  - `coststress`: spread `0.2`, slippage `0.1`, execution delay `1`

The new `scripts/experiments/holding_max_grid.py` keeps the full merged prediction frame available during monthly backtests. This prevents positions near month end from being forced flat only because predictions were filtered to `dataset_month == target_month`.

## Coverage Caveat

| month | evaluation rows | post-exit rows | has post-exit predictions |
|---|---:|---:|---|
| 2025-01 | `28819` | `0` | `false` |
| 2025-02 | `27441` | `0` | `false` |
| 2025-03 | `28972` | `1381` | `true` |
| 2025-04 | `28948` | `1381` | `true` |
| 2025-05 | `30147` | `61` | `true` |
| 2025-06 | `28889` | `1381` | `true` |
| 2025-07 | `31499` | `1320` | `true` |
| 2025-08 | `28970` | `0` | `false` |

This means the 8-month result is useful as a same-input artifact comparison, but it is not a perfect month-end continuation dataset for all months. The script now writes `prediction_coverage.csv`, rejects duplicate `decision_timestamp`, and has `--require-post-coverage` for fresh single-window checks. After this run, `covers_full_max_exit_window` was tightened so it cannot be true when there are no post-exit rows.

## Findings

- `240m` should no longer be treated as the single fixed candidate. The current candidate is the `250..260m` band.
- `260m` is preferred for the next fixed check because it keeps almost all no-cost upside while improving cost-stress total and drawdown versus `240m`.
- `250m` should be carried as a defensive sensitivity because it has much better cost-stress worst month.
- `720m` remains a diagnostic upper-tail setting, not a promoted candidate.
- The monthly prediction-frame rule is now operationalized: do not slice predictions to the evaluation month when a position can exit after month end.
